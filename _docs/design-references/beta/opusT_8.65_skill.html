01 / Foundation

Design Principles

Four convictions that govern every visual decision in KhromaDiet. They are not aspirational — they are constraints.

01

Structural Clarity

Every element declares itself. Borders with conviction, typography with hierarchy, surfaces with purpose. Ambiguity is a bug. If the user has to guess, we failed.

02

Clinical Comfort

Precision without coldness. Sparse layouts breathe confidence, generous whitespace signals trust. Health data deserves calm, not chaos.

03

Earned Presence

Nothing exists without justification. Every component, every color use, every animation must earn its place. Decoration is noise — clarity is signal.

04

Tactile Intelligence

Interactions feel physical. Buttons press, cards lift, toggles snap. Motion reveals system state, not personality. The product thinks; the UI responds.

10 / Motion

Motion Language

Movement is intelligence, not decoration. Three easing curves, four durations. Hover to see each timing in action.

250ms — UI responses
Ease In-Out
400ms — Layout shifts
Spring
250ms — Tactile feedback
Instant
80ms — Micro states
Token Value Use Case
--duration-instant 80ms Color changes, opacity micro-states
--duration-fast 150ms Hover states, focus rings, small toggles
--duration-normal 250ms Button presses, card lifts, tab switches
--duration-slow 400ms Panel reveals, layout transitions, modals
--duration-slower 600ms Chart animations, progress fills, page entrances
--ease-out cubic-bezier(0.16, 1, 0.3, 1) Default for all UI responses
--ease-in-out cubic-bezier(0.65, 0, 0.35, 1) Symmetrical transitions, layout morphs
--ease-spring cubic-bezier(0.34, 1.56, 0.64, 1) Tactile overshoots — buttons, toggles, badges
